================================================================================
                     API ENDPOINTS AVAILABLE
================================================================================

1. POST /api/teacher-requests/submit
   - Teacher submits qualification request
   - Requires: qualifications, experience, contact

2. GET /api/teacher-requests/pending
   - Admin fetches pending requests
   - Support pagination: ?page=1&limit=10
   - Admin-only access

3. POST /api/teacher-requests/approve/:requestId
   - Admin approves a teacher
   - Optional: assignedClasses, assignedSubjects
   - Admin-only access

4. POST /api/teacher-requests/reject/:requestId
   - Admin rejects a teacher
   - Requires: rejectionReason
   - Admin-only access

5. GET /api/teacher-requests/status/:userId
   - Teacher checks their approval status
   - Returns: approvalStatus, request details
